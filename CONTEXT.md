## 작업 워크플로우 (요청 → 구현 → 배포)

> 단일 HTML 프로젝트라 풀스택식 Phase 게이트는 불필요. 대신 **모든 변경에 이 4스텝**을 적용한다.

```
[요청] 섹션 추가·카피 수정·스타일 변경 등
  │
1 · 계획        기획서(노션 SSOT) 해당 항목 대조 → 변경안 + 영향 범위(어느 섹션/토큰) 정리
  │
══ 게이트 · 계획 승인 ══   (이 전엔 코드 안 씀)
  │
2 · 구현        index.html 단일 파일 수정. 디자인 토큰(CSS 변수)·시맨틱 HTML 규칙 준수
  │
3 · 자가검증    아래 "셀프 체크리스트" 0건까지 자가 수정(최대 3회). 안 풀리면 멈추고 원인 보고
  │
4 · 로컬 확인   브라우저로 index.html 열어 데스크탑/모바일(375px) 양쪽 육안 확인
  │
5 · 배포        git push origin HEAD:main → GitHub Pages 자동 갱신 (1~2분 내 반영)
  │
[보고]
```

### 셀프 체크리스트 (3번 자가검증 단계)
- [ ] HTML 구조 유효(닫힘 태그·중첩) · 콘솔 에러 0
- [ ] 모든 색은 CSS 변수(`--color-*`)로만. 하드코딩 HEX 금지
- [ ] 인디고 강조색이 화면의 5~10%만 차지(핵심 숫자·CTA에만) — 남용 시 강조 효과 사라짐
- [ ] 375px 모바일에서 가로 스크롤·잘림·겹침 없음
- [ ] 성과 숫자·경력 연도·회사명이 **노션 기획서와 정확히 일치**(임의 수정 금지)
- [ ] 외부 링크(티스토리·링크드인·메일·PDF) 전부 살아있음 + `target="_blank" rel="noopener"`

## 기술 스택 (확정)

```
HTML5 + CSS3 + Vanilla JS   (단일 index.html, 빌드/번들러/프레임워크 없음)
GitHub Pages                 정적 호스팅 (jaimo012.github.io/<레포명>)
Pretendard (CDN)             헤드라인·본문
JetBrains Mono (CDN)         숫자·라벨 액센트
```

> 외부 의존성은 **폰트 CDN 2개뿐**. JS 라이브러리·프레임워크·빌드 도구 일절 추가하지 않는다(설치하려 하지 말 것). 인터랙션(아코디언·sticky·스크롤)은 순수 JS로 처리.
> 블로그는 **티스토리**에 별도 운영, 랜딩에서 링크로만 연결(자동화·임베드 고려 X).

## 디자인 시스템 (노션 기획서 §2 확정값 이식)

> CSS `:root`에 토큰으로 선언하고 전 영역이 변수만 참조한다.

```css
:root {
  /* 컬러 — 인디고는 화면의 5~10%(핵심 숫자·CTA)에만 */
  --color-bg:        #FAFAF8;  /* 오프화이트 — 배경·여백 */
  --color-text:      #1A1A1A;  /* 차콜 — 본문·헤드라인 */
  --color-accent:    #4F46E5;  /* 인디고 — CTA·핵심 숫자 (강조 전용) */
  --color-accent-dk: #312E81;  /* 딥 인디고 — 소제목·라벨·푸터 배경 */

  /* 타이포 */
  --font-head: 'Pretendard', sans-serif;   /* Bold/ExtraBold */
  --font-body: 'Pretendard', sans-serif;   /* Regular */
  --font-mono: 'JetBrains Mono', monospace;/* 숫자·라벨 */
}
```

**레이아웃 원칙** (기획서 §2)
- 큰 숫자로 성과 강조(결과지향 시각화) · 단단한 그리드 · 당당한 여백
- 2×2 매트릭스 느낌의 구조감을 은은한 모티프로(APM 정신·경영과학 배경과 일관)

**톤앤매너 메시지 위계** (카피·서술 시 우선순위)
1. 실전·결과지향·에너지 (영업/사업개발 DNA) — 최우선
2. 논리·신뢰·묵직함 (정책/MBA 기반)
3. 현대·기술·세련 (AI 기획 지향) — 액센트

> ⚠️ APM(AX Payoff Matrix) 프레임워크 자체는 **전면 노출 금지**. 정신만 메시지·레이아웃에 흡수하고, 전문은 티스토리 링크로만 연결(궁금한 사람만 발견하는 깊이).

## 코드 작성 규칙

1. **단일 파일** — 모든 HTML·CSS·JS는 `index.html` 안에(`<style>`·`<script>`). 외부 파일 분리하지 않는다.
2. **색은 CSS 변수만** — `var(--color-*)`. 컴포넌트에 HEX 직접 입력 금지.
3. **시맨틱 HTML** — `<header>/<main>/<section>/<nav>/<footer>`, 제목 위계(`h1`→`h2`) 준수. div 떡칠 지양.
4. **반응형** — 375px 모바일 우선, 데스크탑까지 깨지지 않게. 단위는 rem/%/clamp 선호, 고정 px 남발 X.
5. **접근성** — `<img>` alt 필수, 링크/버튼에 의미 있는 텍스트, 명도 대비 확보, 연락 아이콘에 `aria-label`.
6. **JS는 최소** — 아코디언·sticky·부드러운 스크롤 정도. 라이브러리 추가 금지. 이벤트는 이벤트 위임 선호.
7. **콘텐츠는 SSOT 대조** — 숫자·연도·회사명·문구는 노션 기획서를 정답지로. 추측해서 채우지 말고, 미정이면 placeholder + 미결정 항목에 남긴다.
8. **성능** — 이미지는 적정 크기로, 폰트는 CDN preconnect. Lighthouse 의식(불필요한 리소스 X).

## 배포 (GitHub Pages)

- 레포에 `index.html`이 루트에 있으면 그게 곧 사이트. 빌드 설정 없음.
- **최초 1회**: GitHub 레포 생성 → Settings > Pages > Source = `main` 브랜치 `/ (root)` 지정 → `https://jaimo012.github.io/<레포명>` 발행.
- **이후 갱신**: `git push origin HEAD:main` → 1~2분 내 자동 반영.
- 커스텀 도메인은 추후 Pages 설정 + `CNAME` 파일로(현재는 `*.github.io` 무료로 충분).

### 배포 전 체크
- [ ] 파일명 정확히 `index.html`(소문자), 루트 위치
- [ ] 이미지 상대경로 정상(`images/...`), 깨진 이미지 없음
- [ ] PDF 파일 레포에 포함 + 버튼 링크 경로 일치
- [ ] 외부 링크(티스토리·링크드인) 실제 주소로 교체(placeholder 잔존 금지)
